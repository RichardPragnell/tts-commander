### 2.1 Tier 1: Fast-Path (Reactive Keyword Match)
* **Target Latency:** $<150\text{ms}$
* **Purpose:** Handles ultra-critical survival and combat inputs.
* **Mechanism:** Bypasses semantic natural language processing models. Plaintext output from the local transcription layer is run against a regular expression and direct lookup table (e.g., matching variations of "Fire", "Shoot", "Blaster", "Stab").

### 2.2 Tier 2: Deep-Path (Semantic Local LLM Router)
* **Target Latency:** $<600\text{ms}$
* **Purpose:** Handles complex, conversational, or indirect commanding phrases.
* **Mechanism:** Plaintext is fed into a lightweight local model (e.g., Llama-3-8B-Instruct or Phi-3-Mini) utilizing a highly optimized, single-shot system prompt. The model extracts an array containing three key variables: `Action`, `Target Room/Node`, and `Urgency/Tone`.

### 2.3 Performance KPIs & Hardware Budgets
* **Audio Sampling Rate:** 16kHz mono-channel input.
* **Max Local VRAM Budget:** 1.5 GB allocated for the STT and small-parameter language processing runtime combined.

## 3. Game Mechanics & Crew Personality Filter Matrix

Once a command passes transcription and intent evaluation, it does not execute instantly. It must first filter through the target crew member's active state and personality variables.

### 3.1 Personality Modifiers Matrix

### 3.2 The AI GM Loop: "But / Because / Therefore"
The overall game state evolves systematically using continuous narrative chains processed by the system's lightweight rules engine:

$$\text{Action Execution} \longrightarrow \text{Consequence Chain}$$

* **Step 1 (Player Input):** *"Fix the hull leak right now, hurry!"*
* **Step 2 (Action Evaluation):** Success. Intern repairs the hull node.
* **Step 3 (AI GM Mutation):** **BUT** *because* the Paranoid Intern dropped his plasma wrench in extreme panic, **THEREFORE** a high-voltage spark has sparked a firespot and generated a microscopic "White Cell Monster" breach in the lower access corridor.
